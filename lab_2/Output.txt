2. ТЕХНИЧЕСКИЕ ПРИЧИНЫ ПРОБЛЕМ

C и Java генераторы:
- Неправильное извлечение битов: (state >> 63) & 1 всегда возвращает 0
- Отсутствие битовой равномерности из-за особенностей переполнения

C++ генератор:
- Параметры LCG не оптимизированы для битовых тестов
- Слишком длинные серии единиц (15 подряд при n=1000)

3. РЕКОМЕНДАЦИИ ПО ИСПРАВЛЕНИЮ

Для C и Java:
Заменить извлечение старшего бита на циклический сдвиг младших битов:

uint64_t bits = state;
for(int i = 0; i < 64; i++) {
    bitstream += (bits & 1) + '0';
    bits = (bits >> 1) | ((state & 1) << 63);
}

Для C++:
Использовать проверенные параметры LCG:
A = 1664525, C = 1013904223, M = 2^32
